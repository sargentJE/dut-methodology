Testing Methodology

This accessibility audit employed a comprehensive approach combining both qualitative user testing and objective technical assessment.

User Testing Protocol

Our user testing methodology involves recruiting participants with diverse disabilities to provide insights that automated testing alone cannot capture. We select participants based on the following criteria:

  • Diversity of Assistive Technologies: We include users of various screen readers (such as NVDA, JAWS, and VoiceOver), magnification software, voice recognition tools, and alternative input devices.
  • Range of Disabilities: Our testing panels typically include individuals with visual, motor, hearing, and cognitive disabilities, as these groups encounter different barriers when using websites.
  • Experience Levels: We incorporate both experienced and novice assistive technology users to understand the full spectrum of user experiences.
  • Demographic Representation: Where possible, we match participant demographics to the target audience of the website under evaluation.

During testing sessions, participants attempt to complete key user journeys that reflect common website interactions and business-critical paths. This task-based approach reveals real-world accessibility barriers that might impact user satisfaction and business outcomes.

We document both successful interactions and barriers encountered, collecting qualitative feedback through think-aloud protocols and post-task interviews. This combination of observational data and participant feedback provides rich insights into the actual user experience that informs our recommendations.

Technical Assessment

Our technical evaluation combines automated and manual testing approaches:

Automated Scanning

We employ industry-standard tools like axe DevTools to identify code-level compliance issues across the WCAG 2.1 AA success criteria.

Manual Expert Review

Our accessibility specialists methodically evaluate elements that automated tools cannot fully assess, including:

  • Semantic structure and logical reading order
  • Keyboard navigation patterns
  • Screen reader announcements and behavior
  • Color contrast and visual presentation
  • Touch target accessibility for mobile devices
  • Logical focus order and visible focus indicators

This dual approach ensures we capture both programmatic violations and experiential barriers that affect users in practice.

Scoring Methodology

Our accessibility scoring system translates complex technical and user experience data into actionable metrics that business stakeholders can easily interpret:

Weighting Framework

The percentage scores are derived using a proprietary weighting system that considers multiple factors:

  1. Conformance Impact: We prioritize WCAG Level A criteria (must-have) higher than Level AA criteria (should-have) to reflect their fundamental importance to basic accessibility.
  2. Functional Impact: Issues that completely prevent task completion receive higher weights than those that merely create inconvenience or inefficiency.
  3. User Population Impact: Barriers affecting larger groups of users with disabilities are weighted more heavily than those affecting smaller populations.
  4. Prevalence Adjustment: Systemic issues appearing across multiple pages receive higher weighting than isolated occurrences.
  5. Business Criticality: Issues affecting key conversion paths (such as forms, checkout processes, or primary navigation) receive additional weighting based on their business impact.

Calculation Process

For each WCAG principle (Perceivable, Operable, Understandable, Robust), we:

  1. Identify all applicable success criteria relevant to the website
  2. Evaluate each criterion against both automated test results and user testing outcomes
  3. Apply our weighting algorithm to account for impact factors
  4. Calculate a normalized percentage score representing the principle's overall compliance level

These principle-specific scores are then used to determine the overall accessibility rating:

Good (80%+)

Few barriers exist; most users with disabilities can successfully use the website

Needs Improvement (50-79%)

Significant barriers exist but can be addressed with targeted improvements

Poor (Below 50%)

Major accessibility barriers prevent many users with disabilities from effectively using the website

This scoring approach correlates strongly with task completion rates from user testing, providing a reliable indicator of both technical compliance and real-world accessibility.
